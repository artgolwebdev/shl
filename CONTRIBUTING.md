# Contributing Guidelines

Thank you for your interest in contributing to the שליחות שוק רמלה project!

## Code Style

### HTML
- Use semantic HTML5 elements
- Maintain proper indentation (4 spaces)
- Add comments for major sections
- Keep attributes on single lines when short, multi-line for long attributes

### CSS
- Use CSS custom properties (variables) for colors and values
- Organize styles by section with clear comments
- Follow BEM-like naming conventions where appropriate
- Group related properties together

### JavaScript
- Use ES6+ features (const/let, arrow functions, template literals)
- Add JSDoc comments for functions
- Keep functions small and focused
- Use descriptive variable names

## File Organization

- Keep HTML, CSS, and JavaScript in separate files
- Use meaningful file names
- Add comments to explain complex logic

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature`
- `fix: Fix bug description`
- `docs: Update documentation`
- `style: Format code`
- `refactor: Refactor code structure`

## Testing

Before submitting changes:
1. Test in multiple browsers
2. Test responsive design on different screen sizes
3. Verify map functionality works correctly
4. Check for console errors

## Questions?

Contact the project maintainer for any questions or clarifications.
